# TsuzuLint ベンチマーク用サンプル文書

本文書は、ベンチマーク計測で利用する日本語技術文書のサンプルである。
いくつかのルールが意図的にヒットするよう設計されており、処理時間を公平に比較できる。

## 概要

TsuzuLintは高速な自然言語リンターであり、Rustで書かれWASMプラグインを採用している。
本プロジェクトのゴールは、textlintと同等の精度を維持しつつ、圧倒的に高速な実行を提供することだ。

## アーキテクチャ

パイプラインはパーサ、AST、ルール実行、キャッシュの四層から成る。
パーサは markdown-rs を利用し、mdast 互換の AST を返す。
AST は bumpalo によるアリーナアロケーションで確保されるため、ノード割り当てが非常に高速である。

### 並列処理

ファイル単位で rayon による並列処理を行うため、複数コアを効率的に活用できる。
大きなリポジトリに対してもスケールし、数千ファイルを数秒でチェックすることが可能である。

### キャッシュ

BLAKE3 によるハッシュベースのキャッシュを採用しており、変更のないファイルは再リントをスキップする。
キャッシュは `.tsuzulint/cache` に JSON Lines 形式で保存される。

## 使い方

インストールは `cargo install --path crates/tsuzulint_cli` で行う。
初期化は `tzlint init`、リントは `tzlint lint "**/*.md"` である。
自動修正は `--fix`、ドライランは `--dry-run` を付与する。

```bash
tzlint lint "**/*.md" --timings
```

## ルール例

### no-todo

ソースコード中の TODO: や FIXME: を検出する。
XXX: もデフォルトでは検出対象だ。

### sentence-length

一文が閾値を超えていないかを検査する単純なルールである。デフォルトは100文字であるが、80文字や120文字などにプロジェクトごとに調整可能であり、文章のスタイルガイドに応じて柔軟に運用できる点が特徴である。

### no-doubled-joshi

日本語の助詞が連続して現れる箇所を検出する。
例えば「私は彼は好きだ」のように「は」が連続する文は、この規則で警告される。
「の」（連体化）や「を」（格助詞）、「て」（接続助詞）は例外として許容される。

### max-ten

一文内の読点の数を制限する。
文中の読点が多すぎると、読点、それは、読み手、にとって、負担、となるため、注意、が、必要である。

## パフォーマンス

本ツールの性能は以下の点で優れている。

- ネイティブバイナリ起動による Node.js ランタイム不要
- WASM プラグインは wasmi もしくは wasmtime 経由で実行
- Parallel Scheduler が CPU コアを飽和させる
- キャッシュによる無駄のないインクリメンタルリント

## 制限事項

以下のような制限事項が存在する。

1. Research-only / WIP であり、API やフォーマットは予告なく変更される。
2. ルールの数が textlint エコシステムより少ない。
3. AI ベースのルールは今後のロードマップで実装予定。
